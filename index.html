Data Tables

Data tables are used to display sets of data. They should be clear, easy to scan, and provide functionality for sorting, pagination, and actions.

Structure and Styling

  • Header: Bold font, aligned with cell content. Use a bottom border to separate from the body.
  • Rows: Use striped rows (`:nth-child(even)`) for better readability. Add a hover state to indicate the active row.
  • Cells: Ensure adequate padding (`px-6 py-4`). Text should be left-aligned, and numerical data right-aligned.
  • Responsiveness: On smaller screens, tables should scroll horizontally.

Modals & Pop-ups

Modals and pop-ups are used to display critical information, confirmations, or forms without navigating away from the current page. They should be used sparingly to avoid interrupting the user flow.

Guidelines

  • Overlay: Use a dark overlay (e.g., `bg-black/50`) to dim the background and focus on the modal.
  • Closing: Modals must be closable via an explicit "Close" or "Cancel" button, the `Esc` key, and by clicking the overlay.
  • Focus: Keyboard focus should be trapped within the modal. The initial focus should be on the most logical element (e.g., the first input field or primary button).

Notifications & Toasts

Notifications and toasts are used to provide timely feedback to users about system status or actions. They should be brief, informative, and non-disruptive.

Guidelines

  • Types: Use semantic colors (Success, Error, Alert, Info) to indicate the nature of the message.
  • Positioning: Toasts typically appear in a corner of the screen (e.g., top-right) and stack vertically.
  • Timing: They should auto-dismiss after a short period (3-5 seconds), but allow users to close them manually.
  • Closing: Include a close button (`X`) for manual dismissal.
